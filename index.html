01 Full-stack · Real-time

QueuePilot

Real-time virtual queue manager for local businesses.

Barbershops and clinics in Costa Rica still run on physical queues. You walk in, grab a number, sit down, and hope your phone battery lasts. Nobody knows their real wait time. Staff can't see the load coming.

QueuePilot is a WebSocket-driven queue system with QR check-in, live wait time estimation, and a three-role dashboard for owners, staff, and customers. No app download. Customers scan a QR code and join from their browser. Staff see the queue update in real time. Owners get analytics: peak hours, average service time, daily throughput.

The hardest part was the state sync. When a customer joins, a staff member calls the next ticket, and an owner is watching the dashboard at the same time — all three views need to update without a page refresh. JWT auth differentiates the roles. FastAPI handles the WebSocket connections. React's state management handles the front-end sync.

  • React 18
  • TypeScript

02 Full-stack

HireFlow

Job application tracker with a Kanban board and a Flask REST API.

Tracking job applications in a spreadsheet works until you have 30 open rows and no way to see what stage they're all in at a glance. HireFlow is a full-stack tracker: Flask REST API, SQLite backend, HTML5 drag-and-drop Kanban board, and a localStorage fallback that kicks in automatically if the API is unavailable. Full CRUD. No setup friction.

03 CLI · Security

LogSentry

CLI security log analyzer with threat pattern detection and 45 passing tests.

Security logs are dense. Reading them manually means missing things. LogSentry is a Python CLI tool that processes log files, detects threat patterns, and outputs structured reports with Rich terminal formatting. Modular threat detection architecture — adding new patterns means adding a module, not editing the core. 45 pytest tests. CI-ready. GitHub only — this one was never meant for a browser.

About

Where this came from.

I started writing Python to automate the tedious parts of my job at Concentrix. Not because I had a plan — because the manual parts were slow and I wanted them done faster. The scripts worked. Then I wanted to know why they worked. That's when it turned into something else.

Every project in this portfolio started with something broken. QueuePilot started with watching people wait in line at a barbershop with no idea how long they'd be there. LogSentry started with a security log that was genuinely hard to read. HireFlow started because job hunting in a spreadsheet falls apart at 30 rows. I build from frustration, not inspiration — which means the things I build have a clear reason to exist.
