Features

Everything a business needs

Not a tutorial CRUD app — each feature solves a real operational problem.

Real-time WebSockets

Queue updates broadcast instantly to every connected client. No polling — pure WebSocket push from FastAPI's async engine.

3-Role Auth System

Owner, staff, and customer roles with separate views and permissions. JWT access + refresh token rotation with bcrypt hashing.

QR Code Check-in

Walk-in customers scan the business QR code to join the queue instantly — no account needed. Generated server-side with python-qrcode.

Analytics Dashboard

Hourly heatmap, weekly trend charts, no-show rate tracking, busiest hours, and top service — all computed server-side.

Smart Wait Estimation

Automatically calculates estimated wait time per customer based on average service duration, updated live as the queue moves.

Service Management

Owners configure services with custom duration and pricing. Staff marks entries through waiting → called → in_service → completed.

Architecture

How it's built

Backend — FastAPIPython

app/routers/

auth · businesses · queue · appointments · analytics

WebSocket Manager

ConnectionManager groups sockets by business slug, broadcasts JSON events on any queue change

Security

JWT access (30 min) + refresh (7 days) rotation · bcrypt · role-based guards

Frontend — React + TSVite

src/hooks/useWebSocket.ts

Custom hook with auto-reconnect + 25s keep-alive ping

src/api/client.ts

Axios interceptor that silently refreshes the JWT on 401 and retries the original request

src/store/authStore.ts

Zustand global store — user state, role-based routing, token persistence
