Background

I went to a barbershop once and waited 45 minutes without knowing if I was next or if three people had skipped ahead of me. No system, just a mental list in the barber's head. I left without a haircut and built this instead.

QueuePilot is the most complete thing I've built — FastAPI backend, React + TypeScript frontend, WebSocket real-time sync, JWT auth with role separation, and a Docker setup. Not a tutorial project. An actual solution to a problem I had.

Features

Not a CRUD app

Each feature came from thinking about how a real barbershop or clinic would actually run this.

Real-time WebSockets

Every queue change broadcasts instantly to every connected client. No polling — pure WebSocket push from FastAPI's async engine. The ConnectionManager groups sockets by business slug and fires JSON events on any state change.

// event pushed to all connected clients
{
  "type": "queue_update",
  "entry_id": 42,
  "status": "called",
  "position": 1,
  "wait_minutes": 0
}

3-Role Auth System

Owner configures services. Staff manages the live queue. Customers check in without an account. JWT access (30 min) + refresh token rotation (7 days), bcrypt hashing, role-based guards on every route.

QR Check-in

Walk-ins scan the shop's QR and join instantly. No account needed. Generated server-side with python-qrcode.

Smart Wait Estimation

Wait time per customer calculated from service duration and queue position. Recalculated live every time the queue moves.

Analytics Dashboard

Hourly heatmap, weekly trend, no-show rate, busiest hours. All computed server-side via SQLAlchemy aggregation queries.
