MoodBite

Mood-based recommender — pick how you feel, get a food, drink, playlist, and ambiance to match. A fictional React front-end project with localStorage.

Source code Live demo

Overview

MoodBite is a fictional front-end project: you pick a mood and it returns a full setup — a food, a drink, a playlist, and an ambiance — with a short reason for the match. Six moods, thirty hand-written recommendation sets. Tag filters use AND logic, so combining "healthy" and "fast" narrows the pool instead of returning almost everything. Favorites and the theme preference both persist in localStorage.

State lives in one place and flows down through props — no router, no state library. The dark mode is a designed palette rather than an inversion, and it applies before React mounts so there's no flash of the wrong theme on reload.

What I Learned

The interesting problems weren't where I expected. Resetting the filters and picking a new recommendation in the same click meant I was reading stale state from a closure until I passed fresh values straight to the helper functions. Hardening localStorage took two layers — a try/catch for private mode and quota limits, plus type guards for when a past session stored the wrong shape. Picking readable text on each mood's accent color became a luminance calculation, so the contrast stays correct if the palette ever changes.
