QueuePilot

Real-time queue management system with WebSocket-driven live updates, JWT authentication, and a React dashboard for operators.

Source code

The Problem

Physical queues waste time for everyone. Customers don’t know when they’ll be served; staff can’t see demand at a glance. Most queue solutions are proprietary, expensive, or require hardware.

What I Built

QueuePilot is a full-stack queue management system built on FastAPI and React 18. A FastAPI backend manages queue state in SQLite via SQLAlchemy and pushes real-time updates to all connected clients over WebSockets. The React frontend gives operators a live dashboard — add customers, call the next person, see wait times update instantly without refreshing.

Authentication uses JWT tokens with refresh token rotation, so operators stay logged in across sessions without compromising security.

Technical Decisions

WebSockets over polling because queue state changes are frequent and latency matters — a 30-second polling interval would make the display useless in a fast-moving environment. The WebSocket connection manager in FastAPI handles multiple simultaneous operator sessions with broadcast-to-all updates.

SQLAlchemy with SQLite was chosen over PostgreSQL for the MVP because the data model is simple (queue → customers → events) and zero infrastructure overhead means it can run on a cheap VPS or even a Raspberry Pi at a physical location.

TypeScript on the frontend catches the shape of WebSocket messages at compile time — one of those cases where the type investment pays off immediately when the API contract changes.

What I Learned

This was my first production-grade WebSocket implementation. The trickiest part was connection lifecycle management — handling disconnects gracefully, cleaning up stale connections, and ensuring queue state re-syncs correctly when a client reconnects after a network drop.
